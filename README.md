## Project Overview

PrepArena is a full-stack, AI-powered platform designed to help software engineers and computer science students rigorously prepare for technical interviews. It combines large language model (LLM) question generation, webcam-based exam proctoring, and an interactive interview simulation environment into a single, cohesive web application.

The platform supports two preparation modes:

| Mode | Description |
|---|---|
| **Mock Test** | Timed, proctored question-answer sessions with MCQ and open-ended text/code questions generated by an LLM. Supports both generic topic-based tests and personalized tests built from the user's own resume and job description. |
| **AI Interview** | Conversational interview simulation. Current version uses a structured 6-question scripted flow. The upcoming version will use real-time voice recognition and dynamic LLM-driven follow-up questions. |

All sessions are stored locally, building a personal history that feeds into the upcoming analytics dashboard, which will visualize progress, strengths, weaknesses, and study streaks over time.

## Key Features

- **LLM-Generated Question Sets** — Groq API (model: `openai/gpt-oss-120b`) generates mixed MCQ + text/code questions tailored to topic, difficulty, and count.
- **Resume + JD Personalization** — Upload your resume (PDF/TXT) and job description to generate custom questions that target the gap between your experience and the role requirements.
- **Real-Time Webcam Proctoring** — OpenCV Haar cascade–based face detection runs in a background thread on the server; JPEG frames are pushed from the browser at 2 fps during tests.
- **Cheating Detection** — Flags face-absent frames, gaze-away frames, and multi-face frames; computes a cheating probability (optionally using an XGBoost model).
- **Intelligent Scoring** — Open-ended text answers are scored using keyword matching, substring matching, and token overlap ratio (Jaccard-style).
- **Keyword Extraction** — An in-browser keyword extractor identifies up to 30 technical skills from uploaded documents using a curated term list and capitalized-phrase heuristics.
- **Session History** — All completed tests and interviews are persisted in `localStorage`, enabling streak calculations, score tracking, and upcoming analytics.
- **Study Scheduler** — A calendar widget on the dashboard lets users schedule upcoming mock tests and interviews.
- **JWT Authentication** — Stateless authentication using `djangorestframework-simplejwt`.
- **ML Data Collection** — A standalone face detection script (`smartFaceDetector.py`) captures per-frame proctoring data to CSV, enabling future model training.

| Component | Technology |
|---|---|

### Architecture Decisions

| Decision | Rationale |
|---|---|
| SQLite for development | Zero-config, portable. Production migration to PostgreSQL is straightforward. |
| JWT (stateless auth) | Decoupled frontend/backend; no server-side session storage required. |
| Groq API for LLM | Low-latency inference via Groq's LPU hardware; dual-key failover for reliability. |
| Push-based proctoring (no server webcam) | Browser captures frames; server never accesses the camera directly. Avoids OS-level permission complexity and enables horizontal scaling. |
| Thread-per-session proctoring | Allows concurrent test sessions without blocking HTTP request handlers. Queue-based frame dropping under load prevents memory pressure. |
| `localStorage` for session history | No backend model required for personal history in the current version; enables offline capability. |
| pdfjs-dist for PDF parsing | Client-side PDF text extraction; no file upload to server for resume/JD content. |

### Mock Test Workflow

```
Dashboard
    │
    ├── [Upload Resume PDF/TXT]
    │       └── extractTextFromFile(file)  (browser, pdfjs-dist or FileReader)
    │               └── extractKeywords(resumeText, jdText)
    │                       └── Stored in PrepContext
    │
    └── [Click "Start Mock Test"]
              │
              ▼
    MockTestSetup
    ├── Standard Mode: Select Topic + Difficulty + Count
    └── Custom Mode:  Select CustomDiff + CustomCount (requires resume/JD)
              │
              ▼ navigate("/dashboard/mock-test", {state: config})
              │
    MockTest — Phase 1: Camera Check
    ├── getUserMedia({video: true})
    │       ├── Granted → show preview → "Begin Test"
    │       └── Denied → "Begin Without Camera" (no proctoring)
              │
              ▼ Phase 2: Test
    ┌─────────────────────────────────────────────────┐
    │  POST /api/proctoring/start/  →  session_id     │
    │  POST /api/generate-mock-test/  →  questions[]  │
    │                                                 │
    │  Frame loop (500ms):                            │
    │    canvas.toBlob(jpeg) → POST /api/proctoring/frame/ │
    │                                                 │
    │  Timer countdown (90s/question)                 │
    │  User answers MCQ / types text                  │
    └─────────────────────────────────────────────────┘
              │ Submit / Timer expires
              ▼
    Score all answers locally
    POST /api/proctoring/stop/ → cheating verdict
    Stop MediaStream tracks
              │
              ▼ navigate("/dashboard/mock-results", {state: results})
              │
    MockTestResults
    ├── Show score ring, proctoring card, Q&A breakdown
    └── addSession() → localStorage (once, on mount)
```

### AI Interview Workflow (Current)

```
Dashboard → [Click "Start AI Interview"]
              │
              ▼
    AIInterviewSetup
    ├── Upload resume (required)
    ├── Upload JD OR enter company + role
    └── [Start Interview]
              │
              ▼ navigate("/dashboard/interview", {state: {company, role, ...}})
              │
    AIInterview
    ├── AI sends Question 1 (scripted, interpolated with company/role)
    ├── User types answer → Send
    ├── AI sends Question 2 ... 6 with simulated delays
    └── On last answer: closing message + session recorded
```

### Proctoring Workflow

```
Browser                              Backend (proctor.py)
────────────────────────────────     ─────────────────────────────────────────
POST /api/proctoring/start/     →    start_session()
                                         └── uuid = str(uuid4())
                                         └── _sessions[uuid] = _Session(uuid)
                                         └── _Session._worker.start()
←  {"session_id": uuid}

[Every 500ms during test:]
canvas.toBlob(jpeg)
POST /api/proctoring/frame/     →    push_frame(sid, jpeg_bytes)
  {session_id, frame (file)}             └── session._q.put_nowait(jpeg_bytes)
←  {"ok": true}                              (drops if queue full — non-blocking)

                                     Background thread (_run):
                                         └── while True: dequeue jpeg
                                         └── cv2.imdecode(jpeg)
                                         └── _analyse_frame(frame, tracked)
                                               ├── Haar cascade: frontal + profile
                                               ├── Skin + aspect-ratio filter
                                               ├── IoU tracking (2-frame confirm)
                                               ├── Eye detection + gaze
                                               └── 4-frame streak for "away"

POST /api/proctoring/stop/      →    stop_session(sid)
  {"session_id": uuid}                   └── Join worker thread
                                         └── result() → compute percentages
                                         └── XGBoost or threshold-based verdict
←  {cheating, flags, pcts, ...}          └── del _sessions[uid]
```
